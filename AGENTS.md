# AGENTS.md

このリポジトリでエージェント（Claude など）が作業する際のガイドラインです。

## PR（プルリクエスト）作成の指針

1. **PR 本文は、ブラウザ拡張機能づくりが初めての人にもわかるよう丁寧に書く。**
   - 「どんな課題を」「どのように解決したか」を、専門用語には補足を添えながら順序立てて説明する。
   - 変更の全体像 → 各ファイルの役割 → 技術的なポイント、の流れでまとめる。

2. **アーティファクト（実際に動く拡張機能）を作成してブラウザにインストールし、動作確認できる場合は、テスト項目と手順を必ず記載する。**
   - インストール方法（例: Firefox の `about:debugging` からの一時読み込み）を具体的に書く。
   - 「何を操作して、何が起きれば成功か」がわかるチェックリスト形式のテスト項目を用意する。
