# 多言語ラベルに関する調査メモ

「おすすめ投稿」「広告（Sponsored）投稿」の判定は、表示言語ごとのラベル文字列を
`src/content/labels.js` の辞書（`IFF_SUGGESTED_LABELS` / `IFF_SPONSORED_LABELS`）に登録し、
**完全一致**で行っている（方式A: DOM）。方式B（JSONインターセプト）はラベル文字列ではなく
JSON 内のフラグ（`explore_story` / `suggested_users` / `is_sponsored` 等）で判定するため、
言語に依存しない。

## なぜ完全一致なのか
- 「おすすめ」「広告」などの短い語は、投稿キャプションに偶然含まれることがある。
- 部分一致だと正当なフォロー投稿を誤って隠す恐れがあるため、**ラベル要素のテキストが
  辞書の語と完全一致**したときだけ隠す。

## 既知の検出漏れ・リスク
1. **未登録の言語**：辞書にない言語で表示している場合、方式A では検出できない。
   - 緩和策：方式B（言語非依存）が有効なら、フィードのおすすめはそちらで除去される。
2. **表記ゆれ**：同じ言語でも UI 更新で文言が変わることがある（例: "Suggested for you" →
   "Suggested for You"）。大文字小文字違いも別エントリとして登録している。
3. **RTL言語（アラビア語など）**：文字列としては登録済みだが、実機での表示文言は
   要確認（本メモ時点では未実機検証）。

## 追加の手順（新しい言語に対応するとき）
1. その言語で Instagram を表示し、おすすめ投稿ヘッダー／広告ラベルの**正確な文字列**を確認する。
2. `src/content/labels.js` の該当辞書に**完全一致の文字列**として追記する。
3. `test/unit/labels.test.js` に検知テストを1行追加する。
